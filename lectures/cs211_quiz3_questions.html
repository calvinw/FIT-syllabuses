CS211 Quiz 3

Quiz Questions

Question 1: Artifacts are documents that an LLM and user collaborate on together through a chat interface.

    1. True
    1. False

Answer: A) True

Question 2: In the artifact workflow, who can send artifact documents?

    1. Both the user and the AI
    1. Only the user
    1. Only the AI
    1. Neither, artifacts are created automatically

Answer: C) Only the AI

Question 3: Artifacts use which system to track document iterations as they’re updated?

    1. Timestamps
    1. Version numbering (version 1, 2, 3, etc.)
    1. Hash codes
    1. File names

Answer: B) Version numbering (version 1, 2, 3, etc.)

Question 4: Which of the following can be created as artifacts?

    1. HTML pages and SVG diagrams
    1. Python and JavaScript code
    1. JSON and CSV data
    1. All of the above

Answer: D) All of the above

Question 5: Artifacts are described using which method in the LLM’s response?

    1. JSON formatting
    1. XML-style tags
    1. Markdown headers
    1. Binary encoding

Answer: B) XML-style tags

Question 6: Custom artifact types can be created for specific use cases like customer order tables or math problem statements.

    1. True
    1. False

Answer: A) True

Question 7: How do you select a column in a dataframe in pandas? Say you have a column called “age” and your dataframe is called customers.

Answer: customers[“age”]

Question 8: “Vibe coding” refers to writing code by describing what you want in natural language to an AI assistant.

    1. True
    1. False

Answer: A) True

Question 9: When you Vibe code you must get everything described perfectly in the first prompt.

    1. True
    1. False

Answer: B) False

Question 10: Which of the following is a benefit of vibe coding?

    1. Faster prototyping
    1. Lower barrier to entry for beginners
    1. Ability to iterate quickly
    1. All of the above

Answer: D) All of the above

Question 11: Google’s Stitch is an AI-powered tool that generates UI designs and HTML/CSS code for web and mobile apps.

    1. True
    1. False

Answer: A) True

Question 12: Stitch can generate working prototypes with full backend functionality and database connections.

    1. True
    1. False

Answer: B) False - Stitch generates the UI part (HTML/CSS) but not the backend functionality

Question 13: Stitch can accept which types of input to generate designs?

    1. Text prompts only
    1. Image uploads only
    1. Both text prompts and images
    1. Neither text nor images

Answer: C) Both text prompts and images

Question 14: Though Stitch can generate HTML/CSS for your pages, you still need to use AI Studio or Gemini further after using Stitch. Why?

Answer: Stitch cannot generate the interactivity for the app, just the look of it.

Question 15: MCP stands for Model Context Protocol.

    1. True
    1. False

Answer: A) True

Question 16: What is the primary purpose of MCP?

    1. To encrypt AI model communications
    1. To enable LLMs to interact with external tools, resources, and services
    1. To train AI models faster
    1. To compress model weights

Answer: B) To enable LLMs to interact with external tools, resources, and services

Question 17: MCP servers can provide which of the following to AI models?

    1. Tools
    1. Resources
    1. Prompts
    1. All of the above

Answer: D) All of the above

Question 18: Which of the following is an advantage of MCP over traditional tools?

    1. Standardization
    1. Discoverability
    1. Interoperability
    1. All of the above

Answer: D) All of the above

Question 19: In the MCP workflow, after the LLM identifies a need for external information, what happens next?

    1. The LLM directly calls the MCP server
    1. The AI App calls the MCP server
    1. The user manually executes the tool
    1. The MCP server automatically runs

Answer: B) The AI App calls the MCP server

Question 20: Dolt is a SQL database with version control features and branches.

    1. True
    1. False

Answer: A) True

Question 21: To create a Dolt database, you need to:

    1. Install software on your local machine
    1. Sign up at dolthub.com and create a database from your profile
    1. Download a database template
    1. Configure a server

Answer: B) Sign up at dolthub.com and create a database from your profile

Question 22: When using an LLM to generate SQL for Dolt, you should have the LLM create:

    1. Separate scripts for CREATE TABLE and INSERT statements
    1. Only CREATE TABLE statements
    1. ONE complete SQL script with both CREATE TABLE and INSERT statements
    1. Only INSERT statements

Answer: C) ONE complete SQL script with both CREATE TABLE and INSERT statements

Question 23: When you upload a SQL script to Dolt, the changes are initially placed on:

    1. The main branch directly
    1. A temporary branch
    1. A backup database
    1. Your local machine

Answer: B) A temporary branch

Question 24: To commit your SQL script changes to the main branch in Dolt, you must:

    1. Upload the file again
    1. Run a command line tool
    1. Create and merge a pull request
    1. Restart the database

Answer: C) Create and merge a pull request

Question 25: A Primary Key is:

    1. The first column in a table
    1. A column that uniquely identifies each row in a table
    1. A column that references another table
    1. An optional field in a database

Answer: B) A column that uniquely identifies each row in a table

Question 26: A primary key value can be NULL.

    1. True
    1. False

Answer: B) False

Question 27: In a Products table with columns: product_id, name, price, in_stock - which column would typically be the primary key?

    1. name
    1. price
    1. product_id
    1. in_stock

Answer: C) product_id

Question 28: A Foreign Key is:

    1. A key from a foreign database
    1. A column that references the primary key of another table
    1. The same as a primary key
    1. A backup key for security

Answer: B) A column that references the primary key of another table

Question 29: Consider these two tables:

Products: product_id (PK), name, price Orders: order_id (PK), product_id, customer, qty

What is the relationship between product_id in these tables?

    1. product_id in Orders is a foreign key referencing Products
    1. product_id in Products is a foreign key referencing Orders
    1. Both are primary keys
    1. There is no relationship

Answer: A) product_id in Orders is a foreign key referencing Products

Question 30: Why would you use a foreign key in the Orders table that references the Products table?

    1. To make the database faster
    1. To maintain relational integrity and link orders to specific products
    1. To save storage space
    1. Foreign keys are not necessary

Answer: B) To maintain relational integrity and link orders to specific products

Question 31: When working with an LLM to edit a Dolt database, it’s recommended to:

    1. Give the LLM direct access to the main branch
    1. Create a separate branch for the LLM to work on
    1. Disable version control
    1. Use a different database system

Answer: B) Create a separate branch for the LLM to work on

Question 32: A Dolt API token can be restricted to work only on specific branches.

    1. True
    1. False

Answer: A) True

Question 33: What is a primary benefit of using Dolt over traditional databases like MySQL?

    1. It’s faster than traditional databases
    1. It uses less storage space
    1. You can track and revert changes to data
    1. It requires no SQL knowledge

Answer: C) You can track and revert changes to data

Question 34: In Dolt, what command is similar to git commit?

    1. dolt save
    1. dolt commit
    1. dolt push
    1. dolt store

Answer: B) dolt commit

Question 35: When using an LLM for data manipulation in Dolt, you should always:

    1. Trust the LLM completely without verification
    1. Review changes before merging them to the main branch
    1. Avoid using version control
    1. Work directly on the main/production database

Answer: B) Review changes before merging them to the main branch

Question 36: In the context of AI-assisted development, what does it mean to “iterate quickly”?

    1. To write code very fast by typing quickly
    1. To make rapid changes and test different approaches easily
    1. To use loops in your code
    1. To deploy code to production immediately

Answer: B) To make rapid changes and test different approaches easily
