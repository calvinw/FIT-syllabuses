Retail Orders Database Schema

Database Schema

This document describes the structure of the retail orders database tables.

Customers Table

Field Type Null Key Default Extra
customer_id int NO PRI NULL auto_increment
first_name varchar(50) NO NULL
last_name varchar(50) NO NULL
email varchar(100) NO UNI NULL

Primary Key: customer_id

Unique Key: email

Description: Stores customer information including name and contact details.

Orders Table

Field Type Null Key Default Extra
order_id int NO PRI NULL auto_increment
customer_id int NO MUL NULL
product_id int NO MUL NULL
order_date timestamp YES CURRENT_TIMESTAMP DEFAULT_GENERATED

Primary Key: order_id

Foreign Keys: - customer_id references customers(customer_id) - product_id references products(product_id)

Description: Records customer orders with references to customer and product. Order date defaults to current timestamp.

Products Table

Field Type Null Key Default Extra
product_id int NO PRI NULL auto_increment
name varchar(100) NO NULL
price decimal(10,2) NO NULL
description varchar(100) NO NULL

Primary Key: product_id

Description: Contains the product catalog with name, price, and description.

Key Relationships

  • One-to-Many: One customer can place many orders
  • Many-to-One: Many orders can reference one product
  • Many-to-Many (through Orders): Customers and products have a many-to-many relationship via the orders table
